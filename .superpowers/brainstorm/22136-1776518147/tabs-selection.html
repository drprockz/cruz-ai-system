Which tabs in the center panel?

Pick all that feel daily-useful. Click multiple. My recommended set is 1, 2, 3, 5 — skip Settings for now (menu in top bar is enough).

1

Conversation ★ default

Live transcript: user turns, CRUZ turns, interleaved tool-progress events. Scrollable history of this conversation. Manual text input at the bottom as a fallback when you can't talk (meeting, subway).

2

Dashboard

Widget grid: Today (calendar events, unread emails, open PRs), Agent metrics (turns/day, tokens, est. time saved), System health (DG / LK / PG / Qdrant / Ollama), Upcoming (6am brief, 3am research, scheduled tasks).

3

Events

Live timestamped stream of everything happening in the system: tool calls, tool results, Sonnet token deltas, webhook hits (GitHub/Vercel/Calendar), agent state changes. Grep-able, filterable by agent/trace. Effectively the agent_logs table as a scrolling console.

4

Agents

Per-agent detail view: what each of the 12 agents has done today, last 10 calls, average latency, token spend, error rate. Clickable to see full agent_logs for that agent. For debugging which specialist is misbehaving.

5

Approvals

Pending approval gates across all agents: TITAN deploys, ECHO sends, REACH outreach, MARK publish, SENTINEL fixes. Click Approve / Deny / Preview. Phase 2 this flow also fires FCM push to phone.

6

Memory

Semantic memory explorer: search past conversations, clicked result loads that conversation into the transcript tab. View procedural preferences (users.preferences). Useful but phase-2 polish — skip for MVP.

7

Tasks

ARQ queue + Plane.so mirror: what's pending, what's running, cron schedule. Essentially a read-only task board. Useful once PM agent is heavily used; skip for MVP.
